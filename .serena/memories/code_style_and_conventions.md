# Code Style and Conventions

## Python
- Follow PEP 8 standards
- Use type hints throughout all functions and classes
- Use Pydantic models for all data structures
- Docstrings for all public functions/classes (Google style)
- Classes use PascalCase
- Functions/variables use snake_case
- Constants use UPPER_SNAKE_CASE

## Pydantic Models
- Use `Field()` for field validation and descriptions
- Use `ConfigDict` for model configuration
- Mark required vs optional fields clearly
- Include examples in field descriptions

## API Design
- RESTful endpoints with clear resource naming
- Proper HTTP status codes
- Consistent error response format
- OpenAPI documentation will be auto-generated by FastAPI

## TypeScript
- Strict mode enabled
- Proper TypeScript types for all components
- Match Python schema structures closely
- Use consistent naming conventions
